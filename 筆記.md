#### 步驟 1：專案初始化

首先，建立一個新的專案資料夾並初始化一個 Node.js 專案。

1.  **建立專案目錄：**
    開啟終端機，並執行以下指令來為專案建立一個新資料夾並進入該資料夾：

    ```bash
    mkdir my-playwright-project
    cd my-playwright-project
    ```

2.  **初始化 pnpm 專案：**
    在現有或新專案中，使用此指令建立一個預設的 `package.json` 檔案來開始管理專案。這是在專案最開始時必須執行的動作，而不是直接使用 `pnpm add`。

    ```bash
    pnpm init
    ```

3.  **設定 ES 模組支援：**
    為利用現代 JavaScript 模組語法 (ES 模組)，可在 `package.json` 檔案中加入一行特定的設定。在 `package.json` 中加入以下這行：

    ```json
    "type": "module"
    ```

---

## 步驟 2：安裝核心依賴套件

在執行完 `pnpm init` 後，接著才使用 pnpm 安裝 Playwright、TypeScript 等開發相關套件。

```bash
pnpm add -D @playwright/test @types/node typescript
```

---

## 步驟 3：使用 Playwright Trace Viewer 追蹤執行情況

Playwright 提供了強大的 Trace Viewer 工具，用來追蹤與除錯測試執行的實際情況。

1.  **設定 Trace 開啟：**
    在 `playwright.config.ts` 中，可將 `trace` 參數設定為 `'on'` 或 `'retain-on-failure'`。
    ```typescript
    use: {
      trace: 'on', // 或 'retain-on-failure'
    },
    ```

2.  **開啟 Trace Viewer 的三種方式：**
    - **VS Code Extension (推薦)：** 在 Testing 面板執行測試後，可直接在編輯器中逐行檢視程式碼與對應的動作。
    - **指令開啟：** 執行測試產出 `trace.zip` 檔後，可透過以下指令開啟：
      ```bash
      pnpm playwright show-trace path/to/trace.zip
      ```
    - **線上版檢視：** 將產出的 `trace.zip` 檔案拖入 [https://trace.playwright.dev/](https://trace.playwright.dev/)。
      > *註：線上版雖然方便，但若未與本機環境連結，有時無法直接點擊程式碼位置進行逐行追蹤。*

## 步驟 4：設定開發環境與 Playwright

1.  **安裝 VS Code Extension：**
    在 VS Code 中搜尋並安裝「Playwright Test for VSCode」擴充功能。這可以讓你直接在編輯器中執行與偵錯測試。

2.  **使用 Playwright UI 介面：**
    - 開啟 VS Code 左側的 **Testing** 面板（燒瓶圖示）。
    - 點擊 **"Run Test"** 旁邊的箭頭，選擇 **"Show Browser"**。
    - 點擊測試案例旁的執行按鈕，即可開啟 Playwright UI 並觀察執行過程。

3.  **設定 TypeScript (`tsconfig.json`)：**
    不需手動執行 `npx tsc --init`，可直接根據專案需求建立 `tsconfig.json`。以下為目前專案使用的推薦設定：

## 步驟 5：進階：Custom Fixture 與 整合 (mergeTests)

Playwright 的 Fixture 系統可以讓我們封裝重複的初始化邏輯，使測試案例更簡潔且易於維護。

### 1. 建立 Custom Fixture (`test.extend`)
可以針對不同需求建立獨立的 Fixture 檔案：
- **Page Objects (`PageObjects.fixture.ts`)**：封裝所有 UI 頁面物件。
  ```typescript
  export const pageObjectTest = baseTest.extend<PageObject>({
    loginPage: async ({ page }, use) => {
      await use(new LoginPage(page));
    },
  });
  ```
- **API Clients (`ApiObjects.fixture.ts`)**：封裝 API 請求與**自動清理 (Teardown)** 邏輯。
  ```typescript
  bookingApiWithDataDeletedAfterward: async ({ bookingApi }, use) => {
    const createResponse = await bookingApi.createBooking(newBookingPayload);
    const bookingId = (await createResponse.json()).bookingid;
    await use({ bookingApi, bookingId });
    // 測試結束後自動刪除資料 (Teardown)
    await bookingApi.deleteBooking(bookingId);
  },
  ```
- **測試資料 (`TestData.fixture.ts`)**：根據環境變數 (`process.env.ENV`) 提供對應的測試資料。

### 2. 鏈結與整合 Fixture (`mergeTests`)
當專案中有多個 Fixture 檔案時，可以使用 `mergeTests` 將它們整合在一個 `test` 物件中，方便測試案引用。
- **範例 (`chainFixtures.ts`)**：
  ```typescript
  import { mergeTests } from '@playwright/test';
  import { pageObjectTest } from './PageObjects.fixture';
  import { dataTest } from './TestData.fixture';

  // 整合 UI 與 Data Fixtures
  export const test = mergeTests(pageObjectTest, dataTest);
  ```

### 3. 在測試中使用整合後的 Fixture
撰寫測試案時，只需從 `chainFixtures.ts` 匯入 `test`，即可同時獲得所有已定義的 Fixtures：
```typescript
import { test, expect } from '../fixtures/chainFixtures';

test('範例測試', async ({ loginPage, standardUserData }) => {
  await loginPage.login(standardUserData.username, standardUserData.password);
  // ...
});
```
